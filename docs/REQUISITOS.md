# ANALISE DE REQUISITOS

## • REQUISITOS FUNCIONAIS
1. O sistema deve permitir o cadastro de diferentes perfis de usuário, do tipo aluno, coach, gestor e estagiario.
2. O sistema deve permitir que o usuário escolha mais de um tipo de perfil
3. O usuário de tipo gestor deve ser cadastrado pelo administrador do sistema
4. Apenas usuários de tipo coach podem cadastrar treino
5. Alunos de tipo aluno devem cadastrar seus resultados diariamente
6. O sistema deve lembrar os usuários de tipo aluno de cadastrar os treinos
7. O sistema deve solicitar numeros de CREF para cadastrar um usuario como coach
8. O usuario de tipo coach deve se afiliar a um ou mais box.
9. O sistema deve enviar a solicitação de afiliação ao gestor do box
10. O sistema deve mostrar analises, alertas, insights, destaques e recomendações aos usuarios, diferenciado por tipo. 
11. Para que sejam fornecidas as informações, o usuario de tipo aluno precisa ter pelo menos 5 registros de treino.
12. O sistema deve permitir que o aluno registre um resultado com adaptações do treino original
13. O sistema deve mostrar ao usario de tipo aluno qual foi o professor responsavel pela criação do treino
14. O sistema deve permitir o usuario do tipo aluno a demonstrar interesse em treinar em um dado horario e ver quem será o professor do horário
15. O sistema deve permitir que os usuarios do tipo estagiario e coach vejam quais alunos tem interesse na turma de um dado horario
16. O sistema deve permitir que usuarios do tipo coach cadastrem movimentos e nomes de seções de treino inexistentes no banco de dados 
17. O sistema deve enviar analises, alertas, insights, destaques e recomendações aos coachs sobre os treinos cadastrados
18. O sistema deve permitir comparar treinos e ciclos 
19. O sistema deve permitir que um usuario do tipo aluno cadastre objetivos
20. O sistema pode sugerir objetivos e adaptações ao usuario tipo aluno
21. O sistema deve usar dados cadastrados e registros de treinos dos alunos de um dado box para avaliar os treinos dos coachs desse mesmo box

## • REQUISITOS NÃO FUNCIONAIS
